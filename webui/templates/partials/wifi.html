{# ---- scan window ---- #}

How long airodump-ng runs each pass before the scanner processes results. Longer = more APs seen on congested 5 GHz channels but slower reaction to new networks. 30 s is a good default for most urban environments. Range: 10–120.

{# ---- channel hop interval ---- #}

Time spent on each channel before moving. Shorter hops = better coverage but lower per-channel dwell (may miss slower beacons). 2 s is a balanced default for 2.4 GHz. Range: 1–10.

Daemon picks up changes on next restart.
